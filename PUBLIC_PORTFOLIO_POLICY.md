# Public Portfolio Policy

This repository is public-facing or public-shareable.
Everything added here must pass a sanitization check first.

## Never Publish

Do not publish any of the following:
- internal runbooks copied verbatim
- company-confidential documents
- customer names when they are not already public and relevant
- internal hostnames, URLs, IPs, ticket numbers, or environment names
- architecture diagrams that reveal internal systems
- screenshots containing sensitive dashboards or private data
- proprietary code or scripts owned by an employer
- personal data, private emails, phone numbers, addresses, or account details
- raw exports from Notion, OneDrive, Gmail, or internal tooling

## Safe To Publish

These are usually safe when rewritten or curated:
- sanitized case studies
- public-safe diagrams rebuilt from scratch
- generalized process improvements
- tooling categories and technology areas
- high-level architecture patterns
- problem/solution/outcome summaries
- open-source code you wrote and are allowed to share
- resume-ready accomplishment statements

## Redaction Rules

When converting private work into public portfolio content:
- replace company-specific details with role-appropriate general language
- remove client names unless they are public and necessary
- remove private identifiers, dates, IDs, and links
- generalize sensitive quantities when exact values are not safe to disclose
- round counts, savings, or scope figures when exact values would expose unnecessary internal detail
- describe outcomes honestly without revealing confidential operations
- preserve technical truth while reducing exposure

## Rewrite Standard

Prefer this structure:
- Context
- Problem
- Constraints
- Actions
- Result
- Tools / Methods
- Public-safe takeaway

## Resume Alignment Rule

Public portfolio entries should support the same core experience model used in the resume system:
- grounded in real work
- chronologically accurate
- technically honest
- non-duplicative

## Decision Rule

If you hesitate because something may be confidential:
- do not upload the original
- rewrite it as a sanitized case study instead

When in doubt, keep it private.
