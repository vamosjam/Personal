## Public-Safe Sharing Standard

Everything here is intentionally sanitized.
I do not publish employer-confidential information, raw internal documents, customer-sensitive details, credentials, internal architecture details, or proprietary material.

If a project cannot be shared directly, I rewrite it as a case study that preserves the technical truth while removing sensitive context.

See [PUBLIC_PORTFOLIO_POLICY.md](./PUBLIC_PORTFOLIO_POLICY.md).

For interviews, I also maintain a separate private evidence kit with sanitized walkthroughs, rebuilt diagrams, and recognition-backed examples that are not appropriate for public GitHub publication.

## Portfolio Principles

- Ground everything in real work.
- Keep chronology and responsibility accurate.
- Prefer evidence and clarity over inflated language.
- Use public-safe summaries instead of raw internal artifacts.
- Avoid duplicate examples when one stronger example says it better.

## Current Scope

This public repo is the showcase layer.
The larger career archive, intake pipeline, and supporting source materials remain private.
Only selected, rewritten, shareable material belongs here.
